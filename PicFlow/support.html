🚀 Quick Start

1

Choose Storage Platform

Open PicFlow settings and choose GitHub, Alibaba Cloud OSS, or Tencent Cloud COS as your image host under "Storage Provider".

2

Configure Storage Info

GitHub: Enter Personal Access Token, repository name, and branch. Alibaba Cloud OSS: Enter AccessKey ID / Secret, Bucket name, Region, and optional custom domain. Tencent Cloud COS: Enter SecretId / SecretKey, Bucket name, Region, and optional custom domain.

3

Test Connection

Click the "Test Connection" button to verify your configuration. Once successful, you can start uploading.

4

Start Uploading

After taking a screenshot, PicFlow auto-detects and prompts upload. You can also drag images into the menu bar popover, or use a global hotkey to upload clipboard images. Links are auto-copied after upload.

❓ FAQ

Upload failed with "Authentication failed"

+
⚠ Configuration

Please check the following:

  1. GitHub: Confirm your Token starts with ghp_ and hasn't expired. Ensure the Token has repo permission.
  2. Alibaba Cloud OSS: Confirm AccessKey ID and Secret are correct. Check if the RAM user has OSS write permissions.
  3. Tencent Cloud COS: Confirm SecretId and SecretKey are correct. Check if the CAM user has COS write permissions.

Upload failed with "Bucket does not exist"

+
⚠ Configuration

Please check:

  1. Whether the Bucket name is spelled correctly (case-sensitive)
  2. Whether the Region matches the Bucket's location (e.g., oss-cn-hangzhou or ap-guangzhou)
  3. Whether the Bucket has been created and is in normal status

No upload prompt after taking a screenshot

+
⚠ Permissions

Auto screenshot detection requires the following:

  1. Ensure PicFlow is running (icon visible in menu bar)
  2. Confirm "Auto Screenshot Detection" is enabled in settings
  3. PicFlow needs Accessibility permission: Go to System Settings → Privacy & Security → Accessibility, and enable PicFlow

Hotkey can't trigger upload

+
⚠ Permissions

Global hotkeys require Accessibility permission. Please go to:

System Settings → Privacy & Security → Accessibility

Find PicFlow in the list and enable it. If already enabled but still not working, try toggling it off and on again. Also check for conflicts with other apps' shortcuts.

Upload succeeded but image link is inaccessible

+
⚠ Network

Please check based on your storage platform:

  1. GitHub: The repository must be set to Public for images to be directly accessible via link
  2. OSS / COS: Confirm the Bucket's read permission is set to "Public Read" or CDN is configured
  3. If using a custom domain, confirm DNS resolution and CNAME are properly configured

How to switch the app's interface language?

+
ℹ Feature

PicFlow supports 39 languages. By default it follows the system language, but you can manually switch in settings:

Settings → Language → Interface Language

Some interface elements may need to be reopened after switching.

Does the app upload my data to third-party servers?

+
🔒 Privacy

No. PicFlow runs locally on your device. All images are uploaded directly to your own configured cloud storage (GitHub / OSS / COS), never through any third-party servers. Credentials are securely stored via macOS Keychain.
