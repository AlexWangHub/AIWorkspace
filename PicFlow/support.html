🚀 快速开始

1

选择存储平台

打开 PicFlow 设置，在「存储服务商」中选择 GitHub、阿里云 OSS 或腾讯云 COS 作为你的图床。

2

配置存储信息

GitHub：填入 Personal Access Token、仓库名、分支名。
阿里云 OSS：填入 AccessKey ID / Secret、Bucket 名称、Region 和可选的自定义域名。
腾讯云 COS：填入 SecretId / SecretKey、Bucket 名称、Region 和可选的自定义域名。

3

测试连接

点击「测试连接」按钮验证配置是否正确。成功后即可开始上传。

4

开始上传

截图后 PicFlow 自动检测并提示上传；也可以将图片拖入菜单栏弹窗，或使用全局快捷键上传剪贴板中的图片。上传完成后链接自动复制到剪贴板。

❓ 常见问题

上传失败，提示「认证失败」

+
⚠ 配置相关

请检查以下几点：

  1. GitHub：确认 Token 以 ghp_ 开头且未过期，确保 Token 拥有 repo 权限
  2. 阿里云 OSS：确认 AccessKey ID 和 Secret 正确，检查 RAM 用户是否拥有 OSS 写入权限
  3. 腾讯云 COS：确认 SecretId 和 SecretKey 正确，检查 CAM 用户是否拥有 COS 写入权限

上传失败，提示「Bucket 不存在」

+
⚠ 配置相关

请检查：

  1. Bucket 名称是否拼写正确（注意大小写）
  2. Region 是否与 Bucket 所在区域一致（例如 oss-cn-hangzhou 或 ap-guangzhou）
  3. Bucket 是否已经创建并处于正常状态

截图后没有自动弹出上传提示

+
⚠ 权限相关

截图自动检测需要以下条件：

  1. 确保 PicFlow 正在运行（菜单栏可见图标）
  2. 在设置中确认「截图自动检测」已开启
  3. PicFlow 需要「辅助功能」权限：前往 系统设置 → 隐私与安全性 → 辅助功能，启用 PicFlow

快捷键无法触发上传

+
⚠ 权限相关

全局快捷键需要「辅助功能」权限。请前往：

系统设置 → 隐私与安全性 → 辅助功能

在列表中找到 PicFlow 并启用。如果已启用但仍不生效，可以取消勾选后重新勾选。也请检查是否与其他应用的快捷键冲突。

上传成功但图片链接无法访问

+
⚠ 网络相关

请根据存储平台检查：

  1. GitHub：仓库需要设置为 Public 才能通过链接直接访问图片
  2. OSS / COS：确认 Bucket 的读权限设置为「公共读」或已配置 CDN
  3. 如果配置了自定义域名，确认 DNS 解析和 CNAME 已正确设置

如何切换 App 界面语言？

+
ℹ 功能相关

PicFlow 支持 39 种语言。默认跟随系统语言，也可以在设置中手动切换：

设置 → 语言设置 → 界面语言

切换后部分界面需要重新打开才能生效。

App 是否会上传我的数据到第三方服务器？

+
🔒 隐私相关

不会。PicFlow 仅在你的设备本地运行，所有图片直接上传到你自己配置的云存储服务（GitHub / OSS / COS），全程不经过任何第三方服务器。密钥信息通过 macOS Keychain 加密存储。
