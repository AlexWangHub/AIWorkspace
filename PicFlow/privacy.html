隐私政策

最后更新日期：2026 年 3 月 23 日

🛡️ PicFlow 不收集、不存储、不传输任何用户个人数据到第三方服务器。你的隐私是我们的首要考量。

概述

PicFlow（以下简称「本应用」）是一款 macOS 菜单栏图床工具，帮助用户将图片上传到云存储平台（GitHub、阿里云 OSS、腾讯云 COS）并获取图片链接。本隐私政策说明了本应用如何处理你的信息。

数据收集

本应用不收集任何个人信息。

具体而言：

  • 不收集用户身份信息（姓名、邮箱、设备标识符等）
  • 不收集使用分析数据或行为数据
  • 不使用任何第三方分析或追踪 SDK
  • 不包含广告或广告追踪
  • 不使用 Cookie 或类似追踪技术

本地存储的数据

本应用会在你的设备本地存储以下信息，这些数据仅保存在你的 macOS 系统中，不会上传到任何服务器：

数据类型 用途 存储方式
云存储密钥（API Token / AccessKey / SecretKey） 连接你的云存储服务 macOS Keychain（加密）
存储桶配置（Bucket、Region、域名等） 指定图片上传的目标位置 SwiftData 本地数据库
快捷键设置 记录用户自定义的全局快捷键 macOS UserDefaults
上传历史记录 显示已上传图片的链接和缩略图 SwiftData 本地数据库
图片压缩设置 记录用户的图片压缩偏好 macOS UserDefaults
语言偏好 记录用户选择的界面语言 macOS UserDefaults

网络通信

本应用仅与你配置的云存储服务通信：

  • GitHub API（api.github.com）— 当选择 GitHub 作为图床时
  • 阿里云 OSS（*.oss-*.aliyuncs.com）— 当选择阿里云 OSS 作为图床时
  • 腾讯云 COS（*.cos.*.myqcloud.com）— 当选择腾讯云 COS 作为图床时

所有网络请求均使用 HTTPS 加密传输。本应用不会与任何其他服务器或第三方服务通信。

💡 你的图片从你的设备直接上传到你自己的云存储服务，中间不经过任何第三方代理或中转服务器。

剪贴板与截图访问

本应用会在以下情况下访问系统剪贴板和截图：

  • 当你按下上传快捷键时，读取剪贴板中的图片
  • 当开启截图自动检测时，监听系统截图事件
  • 当你打开菜单栏弹窗时，显示剪贴板中的图片预览

本应用不会将剪贴板内容发送到任何第三方服务器。截图检测功能可在设置中随时关闭。

辅助功能权限

本应用需要 macOS 辅助功能（Accessibility）权限来注册全局快捷键。该权限仅用于监听键盘快捷键事件，不用于其他任何目的。

第三方服务

本应用使用以下第三方云存储服务的官方 API 来上传图片：

  • GitHub REST API
  • 阿里云对象存储 OSS
  • 腾讯云对象存储 COS

当你使用本应用上传图片时，相关数据将受到对应平台的隐私政策和服务条款约束。

除此之外，本应用不集成任何第三方 SDK 或服务。

儿童隐私

本应用不针对 13 岁以下的儿童。本应用不会有意收集任何儿童的个人信息。

隐私政策变更

如本隐私政策有重大变更，我们将通过应用更新或本页面进行通知。建议你定期查看本页面以了解最新的隐私政策。
